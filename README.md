## 🔀 PR 규칙

1. **브랜치명**: `{github-id}/{일자}-{문제-slug}`
   - 예: `hun425/07-07-two-sum`
2. **PR 제목**: `[YYYY-MM-DD] {문제명} - {github-id}`
3. **PR 본문**: 위 풀이 템플릿 내용 그대로 포함
4. **리뷰 기준**
   - 정답 여부보다 **"다른 풀이 방법이 있는지", "시간/공간복잡도를 더 줄일 수 있는지"** 위주로 코멘트
   - 못 풀었어도 괜찮음 — 시도한 코드와 막힌 지점을 정리해서 PR 올리기
5. **3명 Approve** 받으면 merge
6. 리뷰는 되도록 **24시간 이내** 완료를 목표로 함
7. ** 수동 머지 금지 ** (조건을 만족하면 자동 merge 됨, conflict 나면 해결 후 push 하면 됨)

## 태그 규칙

PR을 merge 하기전 태그가 필수로 붙어있어야합니다.

1. 풀이 성공일 경우 SUCCESS
2. 풀이 실패일 경우 FAILED (고민하다 포기한 경우)
3. 계속 도전해 봤는데 도저히 모르겠어서 도움받고 다시 풀어보고 싶은 경우(HELP)

HELP의 경우에는 같은 문제로 다음 주차에 풀이하는 것을 1회 허용합니다. (단, 전주와 비교했을때 풀이의 내용 및 접근 방식이 달라져야합니다.)

풀이 pr 제출 기한은 해당 주의 일요일 23:59분 까지입니다.

상대 pr 리뷰 기한은 다음 주의 화요일 23:59분 까지입니다.
