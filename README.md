# 🧠 알고리즘 스터디

조윤식의 실력을 나머지 3명이 따라잡는 그날까지 이 스터디는 계속된다.......

## 👥 참여 인원

- 4명

## 📅 진행 방식

- **주 3회** 문제 풀이 (첫 주는 온보딩 기간으로 **주 2회**로 시작 → 2주차부터 정규 운영)
- 문제는 **LeetCode Daily Challenge 우선**
  - 난이도가 너무 높다고 판단되면 자유롭게 다른 문제로 대체 가능 (NeetCode 150, Blind 75, 백준 등 무관)
- 각자 문제를 풀고 **PR로 제출** → 스터디원 중 **최소 1명 이상 Approve** 받아야 merge

## 📁 저장소 구조

```
study-root/
├── README.md
└── members/
    ├── {github-id}/
    │   ├── 2026-07/
    │   │   ├── 07-07-two-sum.md
    │   │   ├── 07-09-valid-parentheses.md
    │   │   └── ...
    │   └── 2026-08/
    │       └── ...
```

- 폴더는 `members/{github-id}/{연-월}/` 구조로 정리
- 파일명은 `{일자}-{문제-slug}.md` 형식 (예: `07-07-two-sum.md`)

## 📝 문제 풀이 파일 템플릿

각 풀이 파일에는 아래 내용을 포함합니다.

```markdown
## [문제명](문제 링크)

### 접근 방법
(어떻게 풀었는지, 왜 이렇게 접근했는지)

### 코드
​```kotlin
// 여기에 코드
​```

### 복잡도
- 시간복잡도:
- 공간복잡도:

### 회고
(막혔던 부분, 더 나은 풀이가 있었는지, 배운 점)
```

## 🔀 PR 규칙

1. **브랜치명**: `{github-id}/{일자}-{문제-slug}`
   - 예: `hun425/07-07-two-sum`
2. **PR 제목**: `[YYYY-MM-DD] {문제명} - {github-id}`
3. **PR 본문**: 위 풀이 템플릿 내용 그대로 포함
4. **리뷰 기준**
   - 정답 여부보다 **"다른 풀이 방법이 있는지", "시간/공간복잡도를 더 줄일 수 있는지"** 위주로 코멘트
   - 못 풀었어도 괜찮음 — 시도한 코드와 막힌 지점을 정리해서 PR 올리기
5. **최소 1명 Approve** 받으면 merge
6. 리뷰는 되도록 **24시간 이내** 완료를 목표로 함
