## [Concatenate Non-Zero Digits and Multiply by Sum II](https://leetcode.com/problems/concatenate-non-zero-digits-and-multiply-by-sum-ii)

### 접근 방법

[3754번](https://leetcode.com/problems/concatenate-non-zero-digits-and-multiply-by-sum-i/description)과 같은 문제이나, 오프라인 배치 쿼리와 자릿수 제한이 커진 버전이다.

1. 자릿수가 10의 5승까지 올 수 있다보니 기본 사칙 연산은 사용할 수 없다. 문제에서 나머지 연산을 요구하므로, 나머지 연산으로 수를 압축하여야 한다.
2. $x$ 값과 $\text{sum}$ 값은 두 범위의 차로서 구할 수 있다는 특징이 있다. 따라서 prefix sum 기법을 적용할 수 있다.

$s = 10203004$
위 경우에서 $l = 2$, $r = 4$ 쿼리를 처리하는 때를 생각해보자. ($s_{l..r} = 203$)
우선은 두 개의 값 $l$, $r$에 대해서 

- $0 \leq t \leq r$ 
- $0 \leq t \leq l - 1$

위 범위를 먼저 계산해보자.

- $x_{0..r} = 123$, $\text{sum}_{0..r} = 6$
- $x_{0..l - 1} = 1$, $\text{sum}_{0..l - 1} = 1$

이 둘에 빼기를 하면 우리가 구하고자 하는 해를 구할 수 있다.

$x_{l..r} = 23$, $\text{sum}_{l..r} = 5$

$x$를 구할 때만 주의하면 되는데, 다음 수식을 따른다는 점이다.

$x_{l..r}=x_{0..r} - 10^{d_{0..r} - d_{0..l}} \times x_{0..l}$ (단, $d_{0..t}$은 0부터 $t$까지의 non-zero 숫자 개수)

위 연산에 쓰이는 10의 거듭제곱꼴, $x_{0..t}$, ${d_{0..t}}$, 그리고 $\text{sum}$을 모두 나머지 연산으로 압축한 뒤 캐싱하면 된다.

### 코드

```kotlin
class Solution {
    fun sumAndMultiply(s: String, queries: Array<IntArray>): IntArray {
        // ready for query batch
        val pow10 = getPow10(s.length)
        val prefixSum = getPrefixSum(s)
        val prefixX = getPrefixX(s)
        val prefixDigits = getPrefixDigits(s)
        
        // do query
        return queries.map { query ->
            query(pow10, prefixSum, prefixX, prefixDigits,query[0], query[1] + 1)
        }.toIntArray()
    }

    fun getPow10(n: Int): List<Long> {
        val pow10 = mutableListOf<Long>(1)

        repeat(n) {
            pow10.add(pow10.last() * 10 % MOD)
        }

        return pow10
    }

    fun getPrefixSum(s: String): List<Long> {
        val prefixSum = mutableListOf<Long>(0)

        s.forEach { c ->
            prefixSum.add((prefixSum.last() + c.digitToInt()) % MOD)
        }

        return prefixSum
    }
    
    fun getPrefixX(s: String): List<Long> {
        val prefixX = mutableListOf<Long>(0)

        s.forEach { c ->
            val digit = c.digitToInt()
            val last = prefixX.last()
            prefixX.add(if (digit > 0) (last * 10 + digit) % MOD else last)
        }

        return prefixX
    }

    fun getPrefixDigits(s: String): List<Int> {
        val prefixDigits = mutableListOf<Int>(0)

        s.forEach { c ->
            prefixDigits.add(prefixDigits.last() + if (c != '0') 1 else 0)
        }

        return prefixDigits
    }

    fun query(
        pow10: List<Long>,
        prefixSum: List<Long>,
        prefixX: List<Long>,
        prefixDigits: List<Int>,
        l: Int,
        r: Int,
    ): Int {
        val x = (prefixX[r] - prefixX[l] * pow10[prefixDigits[r] - prefixDigits[l]] % MOD).mod(MOD)
        val sum = (prefixSum[r] - prefixSum[l]).mod(MOD)

        return (x * sum).mod(MOD).toInt()
    }

    companion object {
        private const val MOD = 1_000_000_007L
    }
}
```

### 복잡도

- 시간복잡도: $O(n + q)$ ($n$은 $s$의 자릿수, $q$는 쿼리 개수)
- 공간복잡도: $O(n)$

### 회고

처음에는 $x$와 $\text{sum}$가 부분해로부터 전체해를 구할 수 있다는 특성만 가지고 세그먼트 트리 방식을 채택했었으나, 그럴 필요 없다는 사실을 알게 되고 더 선형적인 방법으로 전환하였다. 소 잡는 칼 먼저 집는 습관을 고쳐야 한다.
