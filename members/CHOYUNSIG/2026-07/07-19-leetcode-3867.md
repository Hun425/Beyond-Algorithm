## [Sum of GCD of Formed Pairs](https://leetcode.com/problems/sum-of-gcd-of-formed-pairs/)

### 접근 방법

문제가 요구한 대로 순서대로 알고리즘을 구성하면 풀리는 문제이다. GCD를 구하는 알고리즘인 유클리드 호제법을 알고있는지를 체크하는 문제같지만, 파이썬을 사용하는 사용자에게는 `Math.gcd` 함수를 제공하기 때문에 언어 차이가 풀이 차이로 이어지는 문제이다.

### 코드

```kotlin
import kotlin.math.max

object GcdUtil {
    tailrec fun gcd(a: Int, b: Int): Int {
        return if (b == 0) a else gcd(b, a % b)
    }
}

class Solution {
    fun gcdSum(nums: IntArray): Long {
        val sortedPrefixGcd = getPrefixGcd(nums).sorted()

        val pairs = List(size = sortedPrefixGcd.size / 2) { i ->
            val smallest = sortedPrefixGcd[i]
            val largest = sortedPrefixGcd[sortedPrefixGcd.lastIndex - i]

            smallest to largest
        }

        return pairs.sumOf { pair ->
            GcdUtil.gcd(pair.first, pair.second).toLong()
        }
    }

    fun getPrefixGcd(nums: IntArray): IntArray {
        val result = mutableListOf<Int>()

        var maxNum = 0

        nums.forEach { num ->
            maxNum = max(maxNum, num)
            result.add(GcdUtil.gcd(num, maxNum))
        }

        return result.toIntArray()
    }
}
```

### 복잡도

- 시간복잡도: 모든 리스트 원소에 대해 GCD를 수행하므로, $O(n\log{|a|})$, (단, $|a|$는 리스트의 원소가 될 수 있는 정수의 범위 비례값)
- 공간복잡도: 함수 호출 스택을 고려하면 $O(n + \log{|a|})$
