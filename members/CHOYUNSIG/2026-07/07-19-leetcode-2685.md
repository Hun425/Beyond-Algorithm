## [Count the Number of Complete Components](https://leetcode.com/problems/count-the-number-of-complete-components/)

### 접근 방법

그래프에서 클리크의 개수를 구하는 문제이다. 다행이 $n$의 범위가 작아 naive 방법론으로 쉽게 해결이 가능하다.

[클리크](https://ko.wikipedia.org/wiki/%ED%81%B4%EB%A6%AD_(%EA%B7%B8%EB%9E%98%ED%94%84_%EC%9D%B4%EB%A1%A0))란 모든 노드 쌍이 모두 연결된 그래프를 뜻한다.

DFS로 1회 탐색하면서 방문한 모든 노드의 그룹 $G$에 대해 다음이 성립하면 클리크다.
> 임의의 노드 $A (\in G)$가 $(G\text{의 크기} - 1)$ 개의 노드와 연결되어 있다.

모든 노드를 탐색해 위를 체크해주면 된다.

### 코드

```kotlin
class Graph(size: Int) {
    fun connect(index1: Int, index2: Int) {
        val (node1, node2) = nodes[index1] to nodes[index2]

        node1.connect(node2)
        node2.connect(node1)
    }

    fun query(): Int {
        var result = 0

        for (node in nodes) {
            if (node.isVisited) continue

            val group = node.dfs()
            if (group.all { node -> node.connectedNodeCount == group.size - 1 }) {
                result++
            }
        }

        return result
    }

    private class Node {
        fun connect(other: Node) {
            nextNodes.add(other)
        }

        fun dfs(): List<Node> {
            val result = mutableListOf<Node>(this)
            isVisited = true

            for (nextNode in nextNodes) {
                if (nextNode.isVisited) continue
                result.addAll(nextNode.dfs())
            }

            return result
        }

        var isVisited = false; private set
        val connectedNodeCount get() = nextNodes.size

        private val nextNodes = mutableListOf<Node>()
    }

    private val nodes = List(size = size) { _ -> Node() }
}

class Solution {
    fun countCompleteComponents(n: Int, edges: Array<IntArray>): Int {
        val graph = Graph(n)

        for (edge in edges) {
            graph.connect(edge.first(), edge.last())
        }

        return graph.query()
    }
}
```

### 복잡도

- 시간복잡도: 일반 DFS 시간복잡도를 추종하므로, $O(N + E)$
- 공간복잡도: $O(N + E)$
