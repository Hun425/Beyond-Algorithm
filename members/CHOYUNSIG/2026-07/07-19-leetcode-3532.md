## [Path Existence Queries in a Graph I](https://leetcode.com/problems/path-existence-queries-in-a-graph-i/)

### 접근 방법

주어지는 정점이 이미 정렬되어서 들어온다. 따라서 입력으로 들어온 각 노드들은 서로 바로 옆 인덱스의 노드와 체인처럼 연결시켰다고 생각할 수 있다.

어떤 두 노드가 연결되었다는 것은 옆 노드를 타고 이동해서 그 노드까지 이동할 수 있느냐가 된다. 따라서 미리 그 행위가 가능한 노드 그룹을 묶어두어 아이디를 부여해두고, 매 쿼리마다 아이디 일치 여부만 검사하면 문제를 빠르게 해결할 수 있다. 

### 코드

```kotlin
import kotlin.math.abs

class Solution {
    fun pathExistenceQueries(
        n: Int,
        nums: IntArray,
        maxDiff: Int,
        queries: Array<IntArray>,
    ): BooleanArray {
        val group = mutableListOf<Int>()

        repeat(n) { index ->
            if (index == 0) {
                group.add(1)
                return@repeat
            }

            val lastGroup = group.last()
            group.add(if (abs(nums[index - 1] - nums[index]) <= maxDiff) lastGroup else lastGroup + 1)
        }

        return queries.map { query ->
            group[query.first()] == group[query.last()]
        }.toBooleanArray()
    }
}
```

### 복잡도

- 시간복잡도: $O(N)$
- 공간복잡도: $O(N)$
